Figma Make Prompt: Improve Design and UI/UX for Mobile Mockups
Design a premium mobile-first application UI that significantly improves the overall design quality, usability, clarity, and visual sophistication of the existing mockups. The goal is to turn a functional prototype into a high-end, polished mobile product with much stronger UI/UX. The app should feel coherent, elegant, modern, and easy to navigate on a phone. Every screen should be redesigned to improve hierarchy, spacing, readability, consistency, and interaction quality.
The product should feel like a serious blockchain-native mobile app with a premium consumer-grade user experience. It should support technically advanced features, but the interface itself must remain highly intuitive. The design should make complex data feel simple. The UI should reduce cognitive load, guide the user naturally, and create a smooth, high-trust experience.
This is not a visual refresh only. It is a full UI/UX improvement pass. The design should improve:
layout structure,
component consistency,
mobile ergonomics,
content hierarchy,
readability,
interaction clarity,
visual rhythm,
and user confidence.
The app should feel like it was redesigned by a top-tier product designer for a polished fintech, Web3, or mission-control style product.

Main UI/UX Goals
The redesigned mobile mockups should achieve the following:
Make the primary action obvious on every screen.
Improve spacing and reduce visual clutter.
Strengthen hierarchy so the user immediately knows what matters.
Create a more premium and modern design language.
Make navigation easier and more predictable.
Use cards, sections, and modular blocks consistently.
Improve the readability of dense technical or blockchain data.
Add more polished interactions, states, and transitions.
Make the product feel trustworthy, calm, and intelligent.
Maintain a mobile-first layout that works naturally on one hand.
The interface should feel like a mobile product that is already close to production quality, not a rough concept.

Design Philosophy
The core design philosophy should be:
simple but powerful,
technical but human,
minimal but expressive,
premium but practical,
data-rich but not overwhelming.
The visual language should reduce friction. Users should not have to think hard about where to tap or what the data means. The UI should act like a guide. It should use strong typography, consistent spacing, visual grouping, and clear affordances to direct attention.
The design should also feel emotionally calm. Even if the product deals with blockchain, swarm systems, mission control, or complex coordination, the UI should be stable, polished, and easy to trust.

Visual Style Direction
Use a modern mobile aesthetic with a refined technical feel.
Style characteristics:
dark mode first,
elegant accent colors,
subtle gradients,
soft glow highlights,
glass or frosted cards where appropriate,
rounded corners,
clean icons,
clear typography,
high contrast text,
restrained motion,
smooth shadows,
and crisp mobile surfaces.
The app should feel premium, not noisy. Avoid overly flashy crypto visuals. Avoid cluttered dashboards. Avoid too many competing colors. Use color intentionally to indicate:
status,
priority,
confidence,
safety,
success,
warning,
and error.
The UI should feel modern and tactile, with enough depth to look alive but not too much decoration.

Mobile-First UX Principles
Optimize all screens for mobile usage.
Required mobile UX principles:
One-hand usability.
Large enough tap targets.
Clear bottom navigation.
Minimal reliance on hover behavior.
Bottom sheets for secondary content.
Expandable cards for detail.
Clear visual separation between sections.
Short labels and concise copy.
Fast scanability.
Easily reachable primary action buttons.
The layout should always support quick comprehension. A user should be able to understand the screen in a glance.

Hierarchy Improvements
The improved UI must have a very strong hierarchy.
Every screen should follow this structure:
Primary status or headline metric
Primary action
Key supporting summaries
Secondary detail sections
Technical or audit-level details behind expansion
This means the most important information should always appear first and most visibly. The primary call to action should not compete with decorative elements. Secondary content should be organized into digestible cards or compact blocks.
Use visual weight carefully:
bigger text for the most important metric,
smaller text for supporting detail,
stronger color for the active state,
muted tones for secondary metadata.

Layout Improvements
Redesign each screen to feel more intentional and structured.
Better layout patterns:
top status card,
summary metrics row,
action section,
main content feed,
supporting detail cards,
persistent bottom navigation.
Avoid vertical walls of content. Break data into logical blocks. Use consistent spacing between sections. Allow each card to breathe. Keep content density manageable on small screens.
Use modular cards so screens can scale elegantly as data changes. Make sure the layout still looks good when there are:
fewer items,
more items,
empty states,
loading states,
or high-volume data.

Typography Improvements
Typography should be one of the most polished parts of the design.
Typography rules:
Use a clear, modern sans-serif font.
Use large numerical styles for balances and status metrics.
Use medium weight for section titles.
Use regular weight for body copy.
Use subtle tracking and spacing for labels.
Keep line length short on mobile.
Avoid overly decorative fonts.
The typography hierarchy should make the app feel more premium and easier to scan. Numeric data should stand out. Labels should support the UI without distracting from it.

Color and Contrast Improvements
Use color in a more disciplined way.
Suggested color usage:
primary brand color for main CTAs,
green for success and active states,
amber for warnings or pending status,
red for critical alerts,
blue or cyan for informational states,
purple or indigo for blockchain or identity states.
Avoid using too many saturated colors in one screen. The visual language should feel cohesive and elegant. Backgrounds should remain dark or neutral enough for content to pop.
Use gradients only as accent treatment, not everywhere. Use glow sparingly, mostly around status states or active cards.

Component Improvements
Refine the core UI components so they feel more consistent and reusable.
Components to improve:
cards,
stat tiles,
bottom nav,
buttons,
toggles,
chips,
status badges,
progress bars,
list rows,
drawers,
modals,
timeline items,
notification cards,
filter pills.
Each component should have:
consistent padding,
consistent rounding,
consistent shadow treatment,
consistent icon sizing,
consistent label behavior,
and consistent active states.
Make sure the entire design system feels unified.

Screen Redesign Guidance
Rework the mobile screens so each one feels better designed and easier to use.
1. Home Dashboard
Improve the dashboard so it instantly shows:
current status,
active mission or current mode,
key wallet or system metrics,
recent activity,
quick actions.
This screen should have the most refined visual hierarchy in the app. The top area should be strong and summary-focused. Secondary cards should be smaller and more compact. The feed should be easy to skim.
2. Detail Views
Any detail view should have:
a strong title,
a concise summary card,
supporting metadata,
a clear action button,
and expandable technical detail.
Detail screens should feel calm and structured, not crowded.
3. Lists
Turn long lists into clean mobile cards with:
clear grouping,
status labels,
left-aligned icons,
right-aligned key values,
and tap affordances.
4. Settings
Make settings organized into grouped sections with:
section headings,
toggles,
short descriptions,
and carefully spaced rows.
Settings should feel much cleaner than a generic app settings page.

Interaction Improvements
The UI should feel more alive and responsive.
Add subtle interactions such as:
card lift on tap,
smooth screen transitions,
animated progress updates,
glowing active states,
sliding drawers,
expandable panels,
toast confirmations,
notification entry motion,
and loading skeletons.
Motion should guide the user, not distract them. It should clarify changes in state. Use animation to reinforce feedback when actions are taken.

State Design
Each screen should have well-designed states.
Required states:
default,
loading,
empty,
error,
success,
warning,
active,
disabled.
State design requirements:
Loading states should use elegant skeletons.
Empty states should include useful guidance.
Error states should be calm and actionable.
Success states should feel rewarding but not overly flashy.
Warning states should be visually distinct without being alarming.
Good UI/UX depends heavily on state handling, so make these states polished and intentional.

Content Design Improvements
The content in the UI should be more concise and user-friendly.
Rewrite copy to be:
shorter,
clearer,
more action-oriented,
more human,
and less technical where possible.
When technical terms are necessary, provide:
a small explanation,
a helper label,
or contextual metadata.
The app should explain itself naturally through layout and copy.

Data Visualization Improvements
If charts or stats are used, improve them for mobile comprehension.
Better mobile chart design:
use compact charts,
keep axes minimal,
prioritize readability,
use short labels,
avoid overcrowding,
highlight only the key data points,
and include short captions.
Charts should help the user make decisions quickly. They should not become decorative clutter.

Navigation Improvements
Improve the navigation to feel more intuitive.
Navigation requirements:
bottom navigation should be clear and persistent,
active state should be obvious,
labels should be readable,
icons should be consistent,
navigation should support up to 5 main areas,
and each route should feel distinct.
If needed, include a floating action button or quick command entry point for the most important actions.

Accessibility Improvements
Make the UI easier for all users to use.
Accessibility considerations:
sufficient contrast,
readable text sizes,
clear touch targets,
consistent icon labels,
no reliance on color alone,
logical focus order,
and easy state recognition.
The app should remain beautiful while also being highly usable.

Trust and Clarity
Because the app likely deals with blockchain, coordination, or mission-like data, the interface should inspire trust.
Trust-building design cues:
clean structure,
visible timestamps,
clear labels,
subtle verification markers,
smooth transitions,
no clutter,
no confusing spacing,
no ambiguous buttons.
Users should always know:
what is happening,
what changed,
what is safe,
and what they can do next.

Final Figma Make Output
Figma Make should generate:
a cleaned-up mobile design system,
more polished screen layouts,
improved hierarchy,
smoother navigation,
better cards and lists,
stronger typography,
better spacing,
more refined state handling,
and a more premium overall feel.
The final result should be a much better mobile product experience that feels modern, technical, polished, and easy to use. The prototype should look significantly more professional than a raw mockup and should be strong enough for demos, investor review, and development handoff.
