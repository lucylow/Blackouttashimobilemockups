Mobile App Design for Tashi Blockchain: Core Product, Wallet, Identity, and Home Experience
Use this prompt in Figma Make as the first half of the mobile app design specification.
Design a premium, futuristic, mobile-first application for the Tashi Blockchain ecosystem. The app should feel like a cross between a modern crypto wallet, a swarm operations dashboard, a decentralized mission control system, and a consumer-friendly mobile product. The interface must prioritize clarity, trust, speed, and blockchain utility, while still feeling visually exciting and highly contemporary. The app should be designed first for iPhone-sized screens, then scale gracefully to Android and larger devices. Make the mobile experience feel native, polished, intelligent, and production-ready.
The app is not a generic wallet. It is a mobile command center for Tashi blockchain users who need to interact with on-chain assets, swarm coordination features, decentralized tasks, identity, staking, governance, transaction history, notifications, and mission status. This means the UI should balance consumer simplicity with advanced blockchain depth. The most important design principle is that the user should never feel lost, even if the underlying system is highly technical.
Create the app as a modular mobile product with a bottom navigation bar and multiple high-quality screens. Every screen should use clean spacing, strong hierarchy, visual feedback, animated micro-interactions, and digestible content blocks. Avoid clutter. Make the screens feel premium and futuristic, with subtle gradients, glassmorphism, soft shadows, and clear data visualization. The app should feel trustworthy, secure, and highly interactive.

Product Vision
The app serves three main goals:
Let users manage Tashi blockchain identity, assets, staking, and transactions from mobile.
Let users observe and participate in swarm-based coordination or missions.
Let users receive notifications, task updates, and blockchain-based state changes in a fast, understandable mobile interface.
The mobile product should feel like a hybrid of:
a crypto wallet,
a decentralized identity app,
a mission control dashboard,
a staking and governance app,
and a swarm coordination tool.
The UI should make Tashi blockchain feel practical and usable, not abstract. Every screen should answer: What can the user do right now? What is happening on-chain? What is the next action?

Visual Design Direction
The app should use a dark, futuristic interface with vivid accent colors and soft luminous gradients. The default theme should be dark mode with optional light mode support, but the primary design should be optimized for dark mode because it better suits blockchain products and gives a premium technical feel.
Use these visual patterns:
Deep charcoal and black backgrounds.
Purple, teal, emerald, and cyan highlights.
Subtle neon edges for active elements.
Frosted glass cards for summaries and action panels.
Thin divider lines and soft glow accents.
Rounded corners with a modern mobile aesthetic.
Crisp iconography with a consistent style.
Small but readable typography.
Strong visual hierarchy for balances, statuses, and mission states.
The visual tone should communicate:
trust,
speed,
intelligence,
connectedness,
and real-time blockchain activity.
Important: do not make it look like a cheap crypto app. Make it look like a high-end mobile operating system for decentralized operations.

Mobile Design Constraints
Design everything with true mobile-first thinking:
All primary actions must be reachable with one thumb.
Use bottom tabs for the most common navigation.
Keep critical status visible near the top.
Support swipeable cards and stacked sections.
Use expandable panels for deeper technical details.
Keep the app readable under motion and daylight.
Ensure touch targets are large and accessible.
Avoid dense tables on mobile unless they are heavily adapted.
Prefer cards, chips, sheets, drawers, and modal bottom sheets.
Show enough data to be useful, but never overwhelm the screen.
All screens should feel like they were designed intentionally for phones, not downsized from desktop.

Navigation Model
Use a bottom navigation bar with 5 main tabs:
Home
Wallet
Swarm
Stake
Profile
Each tab should be clearly labeled with an icon and use active-state highlighting. The navigation should feel native, fast, and consistent. Consider a floating central action button if appropriate for a quick action such as Scan, Deploy, Send, or Join Mission.
The navigation should support the idea that the app is not just a wallet, but a live control surface for blockchain-based swarm behavior.

Screen 1: Launch / Welcome / Authentication
Design a highly polished onboarding and entry flow for new users. The first impression should immediately communicate that the app is secure, intelligent, and blockchain-native.
Required elements:
App logo and Tashi brand mark.
Short value proposition.
Animated background with subtle moving shapes or network lines.
Clear “Get Started” and “Sign In” buttons.
Optional biometric login shortcut.
A short explanation of what Tashi blockchain does.
A security note about wallet ownership and identity.
Suggested copy style:
“Coordinate on-chain. Operate in real time.”
“Your mobile command center for Tashi.”
“Secure identity, staking, swarm tasks, and mission control.”
The onboarding should include 2–4 short slides if needed. Each slide should focus on one concept:
Identity and security.
Blockchain assets and staking.
Swarm coordination and mission updates.
Real-time notifications and governance.
The onboarding must remain lightweight and visually striking. Keep copy concise, but allow for a few explanatory bullets.

Screen 2: Home Dashboard
This is the central landing screen after login. It should act like a mission summary and wallet overview combined into one.
The Home screen should include:
A prominent top summary card.
Current Tashi wallet balance.
Network status indicator.
Recent mission status or swarm status.
Notification count.
Quick action buttons.
A compact “what’s happening now” activity feed.
Recommended components:
Balance card with large numeric display.
Network health chip showing online/offline/synced status.
Staking status with APY or participation summary if relevant.
Mission card showing active mission name, progress, and role.
Quick action row for Send, Receive, Stake, Vote, Join Mission.
Activity timeline showing latest transactions or swarm events.
Design the home dashboard to answer these questions immediately:
How much value do I have?
What is the network doing?
What mission am I part of?
What should I do next?
The home screen should feel highly actionable. It should not be a passive info page.

Screen 3: Wallet Overview
Design a wallet area that looks secure, elegant, and easy to use on mobile.
Required sections:
Main asset balance.
Token list with icons, prices, and changes.
Recent transaction list.
Receive and send buttons.
QR receive code panel.
Transaction status labels.
The wallet screen should support:
Multiple tokens or chain assets.
Expandable token rows.
Horizontal cards for featured assets.
A compact portfolio performance widget.
A “hide/show balance” toggle for privacy.
UX guidance:
The user should be able to glance at the screen and understand:
total value,
individual asset breakdown,
recent activity,
and next action.
Use animated transitions when moving from balance overview to token details. Include a card-style detail sheet for each asset.

Screen 4: Identity and Security
Since blockchain identity is important, create a dedicated identity section.
Screen content:
User avatar or profile identity badge.
Decentralized identity status.
Wallet address summary.
Verified roles or permissions.
Connected devices.
Security score or trust indicator.
Biometrics and recovery controls.
Possible design elements:
“Tashi Identity” card.
“Verified by device” chip.
Wallet address with copy button and QR option.
Recovery phrase or backup warning panel.
Session list with active login devices.
Security alerts panel.
The visual tone should be serious and trustworthy. Use clean typography and security-focused iconography. Make it clear the user owns their identity and keys. Present important warnings in a dignified, non-alarming way.

Screen 5: Notifications Center
Design a mobile notification center for blockchain and swarm events.
Notification types should include:
Transaction confirmations.
Mission updates.
Vote requests.
Relayed signals.
Victim found alerts.
Relay chain changes.
Low battery warnings.
Thermal alerts.
Security events.
Each notification should contain:
Title.
Short descriptive text.
Timestamp.
Severity or priority marker.
Action button where relevant.
Make the notifications feel smart and not spammy. Group them into categories:
On-chain,
Swarm,
Safety,
Governance,
System.
Use swipe gestures or card actions if appropriate. Allow the user to mark items as read, pin important alerts, or open them into deeper event views.

Screen 6: Quick Actions / Floating Command Sheet
Design a quick action UI accessible from anywhere in the app.
Actions should include:
Send funds.
Receive funds.
Stake tokens.
Vote.
Join mission.
Scan QR.
Open mission map.
Trigger emergency retreat if authorized.
Use a bottom sheet or floating action menu that feels fast and native on mobile. The quick action system should feel like a command palette for mobile.
Interaction expectations:
Tap floating button.
Expand action sheet.
Show icons, labels, and short context.
Allow a fast tap to open a corresponding flow.
The purpose of this feature is to make the app feel highly usable and action-oriented.

Screen 7: Stake Summary Preview
Design a compact staking preview even in the first half of the prompt, because staking is core to Tashi blockchain identity.
Include:
Total stake.
Active stake.
Available stake.
APY or participation metric.
Validator or pool relationship.
Recent staking rewards.
“Stake more” button.
The staking area should feel like a financial dashboard but with blockchain-native clarity. Keep it elegant and readable on mobile. Use chips, progress bars, and compact stat cards.

Screen 8: Swarm Status Preview
Even in Prompt 1, introduce a simplified swarm status screen as a teaser for the coordination capabilities.
Include:
Active swarm count.
Mission progress ring.
Agent health summary.
Live network connectivity.
Current role assignment summary.
A simplified map or radar visualization.
This screen should establish that the app is not just a wallet. It is a live command interface for autonomous decentralized agents.
Use a small radar, heat ring, or map card to imply motion and coordination.

Screen 9: Profile and Settings
Create a simple profile/settings screen that supports user control and customization.
Include:
Profile identity.
Display name.
Chain/network preferences.
Notification settings.
Theme settings.
Language settings.
Security settings.
Logout / lock controls.
Make settings feel calm, organized, and secure. Avoid clutter. Use sections with clear labels and consistent iconography.

Screen Behavior and Motion
Design the mobile app with tasteful motion:
Fade and slide between screens.
Bottom sheets should spring upward.
Cards should lift subtly on tap.
Active tabs should animate smoothly.
Status changes should pulse or glow briefly.
Balance updates should animate count changes.
Notification arrival should include a small accent motion.
Motion should reinforce clarity, not distract. Use it to signal state changes and important actions.

Information Hierarchy Rules
Every screen should prioritize content in this order:
Primary state or status.
Main action.
Secondary details.
Supporting metadata.
Deeper technical detail behind expandable controls.
That means:
In Home, the first thing is total balance and mission status.
In Wallet, the first thing is total value and transaction actions.
In Swarm, the first thing is mission progress and current role.
In Stake, the first thing is staking position and reward flow.
In Profile, the first thing is identity and security.

Component Style System
Create a consistent component language across the app:
Large primary cards.
Smaller stat chips.
Glass panels for summaries.
Bottom sheets for extended detail.
Notification cards with severity tags.
Toggle switches for privacy and settings.
Badge chips for roles/status.
Progress rings for mission and staking state.
Timeline rows for activity feeds.
Typography should be highly legible:
Big numbers for balances.
Medium headings.
Short labels.
Small metadata text.
Avoid overly decorative fonts.

Data Presentation Rules
Present blockchain data in a mobile-friendly way.
Best practices:
Use human-readable labels.
Abbreviate large numbers.
Group related stats.
Never show a wall of raw blockchain data.
Hide technical hashes behind copy buttons or expanders.
Show network details only where needed.
Include helpful tooltips or inline context for blockchain terms.

Accessibility and Trust
Because this is a blockchain product, trust is essential.
The app should visibly support:
biometric login,
clear security status,
readable contrast,
high tap targets,
accessible icon labels,
privacy controls,
confirmation dialogs for high-risk actions.
The UI should feel safe even when handling valuable assets and mission-critical swarm events.

Output Expectations for Figma Make
When generating the screens, produce:
a cohesive mobile design system,
multiple polished mobile frames,
reusable components,
a connected flow between screens,
dark-mode first styling,
blockchain-native interface elements,
and a premium futuristic brand feel.
The overall product should feel like a mobile operating system for Tashi blockchain.
